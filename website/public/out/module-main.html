<static> init(path, env, cb)

Main initialization function for the framework.

This is typically the last call in your ca_methods.js file.

It recursively loads and instatiates all the components described in framework.json. When all of them are active, it starts listening for requests at an specified port. When the loader cannot find a module it just throws an exception.

Parameters:
Name Type Argument Description
path string

Directory with json descriptions and modules. In most cases set to __dirname (same directory as your ca_methods.js file).
env Object

Map with properties that override the spec.env of the top component.
cb caf.cb <optional>

An optional callback that will return an error or the context object with a top level sup component (and a loader component). This is mainly used for testing/debugging or to implement custom recovery.

<inner> loadComponent(spec, context, env, secrets, override, cb)

Loads and instantiates a new component.

Parameters:
Name Type Description
spec {module: string, name: string, env: Object}

Configuration data for this component extracted from a JSON description.
context Object

A context to find needed components (like the loader).
env Object

A map with properties that should be mixed with spec.env.
secrets Object

Map with run-time information that should be encapsulated within the new component methods.
override boolean

If true, when there is a conflict because a property has been defined in both env and spec.env, env has priority. Otherwise, we keep the value in spec.env. If there are no conflicts we always merge properties.
cb caf.cb

A callback to return the new component or an error.
